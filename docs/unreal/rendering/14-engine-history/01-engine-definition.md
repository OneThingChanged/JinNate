게임 엔진의 개념과 핵심 구성 요소를 설명합니다.

## 게임 엔진이란?

게임 엔진은 게임 개발팀이 기술 구현보다 게임 콘텐츠에 집중할 수 있도록 하는 일련의 모듈과 인터페이스입니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    게임 엔진 구조                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    Game Content                          │   │
│  │         (스크립트, 에셋, 레벨, AI 로직)                 │   │
│  └───────────────────────┬─────────────────────────────────┘   │
│                          │                                      │
│  ┌───────────────────────┴─────────────────────────────────┐   │
│  │                    Game Engine                           │   │
│  │  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐       │   │
│  │  │Rendering│ │ Physics │ │  Audio  │ │   AI    │       │   │
│  │  └─────────┘ └─────────┘ └─────────┘ └─────────┘       │   │
│  │  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐       │   │
│  │  │Animation│ │Networking│ │  Input  │ │Scripting│       │   │
│  │  └─────────┘ └─────────┘ └─────────┘ └─────────┘       │   │
│  └───────────────────────┬─────────────────────────────────┘   │
│                          │                                      │
│  ┌───────────────────────┴─────────────────────────────────┐   │
│  │              Platform Abstraction Layer                  │   │
│  └───────────────────────┬─────────────────────────────────┘   │
│                          │                                      │
│  ┌───────────────────────┴─────────────────────────────────┐   │
│  │     OS / Hardware (Windows, Console, Mobile, GPU)        │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 설계 원칙

### 데이터 주도 아키텍처

```
┌─────────────────────────────────────────────────────────────────┐
│                    데이터 주도 설계                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  전통적 방식:                                                   │
│  └── 콘텐츠가 코드에 하드코딩됨                                │
│      └── 콘텐츠 변경 = 코드 재컴파일                           │
│                                                                 │
│  데이터 주도 방식:                                              │
│  └── 콘텐츠가 외부 파일로 분리됨                               │
│      └── 콘텐츠 변경 = 파일만 수정                             │
│      └── 에디터에서 실시간 수정 가능                           │
│                                                                 │
│  장점:                                                          │
│  • 아티스트/디자이너가 프로그래머 없이 작업 가능               │
│  • 빠른 이터레이션                                             │
│  • 코드와 콘텐츠의 독립적 버전 관리                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 모듈화

```
┌─────────────────────────────────────────────────────────────────┐
│                    모듈화 구조                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  각 시스템이 독립적으로 동작:                                   │
│                                                                 │
│  ┌──────────────┐     ┌──────────────┐                         │
│  │   Renderer   │←───▶│Scene Manager │                         │
│  └──────────────┘     └──────────────┘                         │
│         ↑                    ↑                                  │
│         │                    │                                  │
│         ▼                    ▼                                  │
│  ┌──────────────┐     ┌──────────────┐                         │
│  │   Resource   │←───▶│    Asset     │                         │
│  │   Manager    │     │   Manager    │                         │
│  └──────────────┘     └──────────────┘                         │
│                                                                 │
│  인터페이스를 통한 느슨한 결합 → 교체/업그레이드 용이           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 렌더링 엔진 구성

| 컴포넌트 | 역할 |
|----------|------|
| **Scene Graph** | 오브젝트 계층 구조 관리 |
| **Visibility System** | 컬링, LOD 선택 |
| **Render Queue** | 드로우 콜 정렬/배치 |
| **Shader System** | 셰이더 관리/컴파일 |
| **Material System** | 머티리얼 파라미터 관리 |
| **Lighting System** | 광원, 그림자 처리 |
| **Post Process** | 후처리 효과 |

## CPU vs GPU

| 특성 | CPU | GPU |
|------|-----|-----|
| 코어 수 | 4-128 | 수천 개 |
| 코어 복잡도 | 고복잡 | 저복잡 |
| 클럭 속도 | 3-5 GHz | 1-2 GHz |
| 캐시 | 코어당 수십 MB | 코어당 수십 KB |
| 분기 처리 | 우수 | 취약 |
| 최적 작업 | 직렬, 복잡한 로직 | 병렬, 단순 연산 |
| 메모리 대역폭 | ~100 GB/s | ~1000 GB/s |

## 핵심 개념

```
┌─────────────────────────────────────────────────────────────────┐
│                    하드웨어 핵심 개념                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  CPU 개념:                                                      │
│  • Pipeline: 명령어를 단계별로 동시 처리                       │
│  • Superscalar: 여러 명령어 동시 실행                          │
│  • Out-of-Order: 의존성 없는 명령어 먼저 실행                  │
│  • Branch Prediction: 분기 결과 예측                           │
│  • Speculative Execution: 예측에 따라 미리 실행                │
│                                                                 │
│  GPU 개념:                                                      │
│  • SIMT: Single Instruction, Multiple Threads                  │
│  • Warp/Wavefront: 동시 실행되는 스레드 그룹                   │
│  • SM/CU: Streaming Multiprocessor / Compute Unit              │
│  • Occupancy: SM 자원 활용률                                   │
│  • Register Pressure: 레지스터 사용량 제약                     │
│                                                                 │
│  메모리 개념:                                                   │
│  • Latency: 접근 지연 시간                                     │
│  • Bandwidth: 전송 대역폭                                      │
│  • Cache: 자주 사용하는 데이터 저장                            │
│  • Coalescing: 메모리 접근 병합                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
